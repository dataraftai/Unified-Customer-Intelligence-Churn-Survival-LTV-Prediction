# 📉 Unified-Customer-Intelligence-Churn-Survival-LTV-Prediction


An end-to-end Machine Learning ecosystem designed for the telecom industry. This project doesn't just predict if a customer will leave; it predicts how much they are worth (LTV) and when the loss will occur (Survival), allowing for data-driven retention strategies.

### 📌 Project Description 

This project solves the "Retention Paradox": Should a company spend money to save every customer? By integrating a Classification model with a Regression model, we can identify "At-Risk" customers and prioritize them based on their financial value to the company.

The system is deployed as a production-ready FastAPI service, containerized with Docker, and features a dual-model pipeline that handles feature engineering and prediction in a single request.

## 🚀 Project Overview
Classification: XGBoost Classifier identifying churn probability.

Regression: XGBoost Regressor predicting Customer Lifetime Value (LTV).

Survival Analysis: Kaplan-Meier modeling to predict the "time-to-churn" horizon.

Business Intelligence: Automatic segmentation of customers into High, Standard, and Low priority based on profitability.

Deployment: Fully containerized REST API via Docker.

## 🧠 Model Performance
1. Churn Prediction (XGBClassifier)
ROC-AUC: 0.85 (Strong ability to distinguish between churners and loyalists).

Recall (Churn=1): 81% (Optimized to ensure we catch as many at-risk customers as possible).

Precision: 53% (A calculated tradeoff to maximize retention coverage).

2. Lifetime Value Prediction (XGBRegressor)
Target: Total Charges (Total revenue generated by the customer).

R² Score: 0.80

Mean Absolute Error (MAE): $727.91

Anti-Leakage Protocol: The Tenure column was removed during training to ensure the model predicts future potential rather than simply calculating tenure multiplied by monthly charges.

## 📊 Business Logic: Profitability & Priority
The API intersects both models to categorize users:

High Priority: High Churn Risk + High LTV (Worth saving with expensive offers).

Low Priority: High Churn Risk + Low LTV (Non-profitable; better to let them churn).
